**[P2-2] `read_text()` can raise `UnicodeDecodeError`, which escapes `run()` unjournalled and un-normalized.**
`runtime.py:208` reads with `read_text()` default (locale) encoding; `UnicodeDecodeError` is a `ValueError`, **not** an `OSError`, so `except OSError` at `runtime.py:209` does not catch it. Concrete scenario: an executor authored under a UTF-8 locale whose source carries a non-ASCII docstring/comment, later opened under a C/POSIX-locale runtime (container/locale drift), or a binary-corrupted/orphan executor file on disk. In both, `read_text` raises `UnicodeDecodeError`, which propagates out of `run()` raw and unjournalled ŌĆö the same class of breach the P1 addressed, but for a different exception family. **Not a regression** (identical at `89572ff`) and narrower (the atomic `write_text` in `registry.write_candidate` never produces such a file in normal flow), hence P2 not P1. Recommend `except (OSError, UnicodeDecodeError)` or read-bytes-then-decode-with-normalized refusal.
