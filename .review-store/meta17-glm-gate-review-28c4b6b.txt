The citing test `test_runtime_stat_failure_is_journalled` (`tests/test_compilation_runtime.py:1304-1389`) **genuinely constrains the guard**: the monkeypatch *replaces* `Path.exists` with a raiser, so on **every** version the guard branch executes (including 3.13+, where the real `exists()` would swallow `PermissionError`). ### P2-1 ‚Äî Pre-existing sibling: unjournaled `ValueError` escape on an embedded-null-byte tampered `registry.path`.
This is the site prior gates missed, but it is **not the `OSError` class** those gates were closing, and it is **pre-existing** (the `resolve`/`exists`/`read_text` guards catch `OSError`[+`UnicodeDecodeError`] only; breadth unchanged by 28c4b6b). The bounded-authority comment at `runtime.py:185-194` explicitly contemplates "a tampered registry.path", so this is in the threat model.

- `executor.missing-source` test (`tests/test_compilation_runtime.py:1281`) constructs `path=f"{pack}/executors/wf-001/absent.py"`; a tampered `registry.json` could instead carry `path` containing `\x00`.
- `resolve(strict=False)` (`runtime.py:198`) on such a path reaches `os.lstat`/`os.readlink`, which raise `ValueError("embedded null byte")` ‚Äî **not** `OSError`, so `except OSError` (`runtime.py:203`) does not catch it; `is_relative_to` is never reached (containment check bypassed) and the raw `ValueError` escapes `run()` unjournaled.

- **Failure scenario:** operator/attacker edits `registry.json` so an active record's `path` contains `\x00`; the run aborts with a raw `ValueError`, no provenance event, no `RuntimeCompError` ‚Äî silent swallow, and the containment check was skipped entirely.

**Disposition:** not a 28c4b6b regression, not the diff's claimed scope ‚Üí **does not block this gate**.
