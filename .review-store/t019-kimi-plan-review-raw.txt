 Notes on probes that resolved clean

 - Line accuracy (probe 1): apart from the P3
   locator errors above, every file:line claim in
   the budget is exact — unusually disciplined.
 - Legacy board operability (probe 4): with the
   chain validator kept out of validateState,
   sync/projection/all transitions on the current
   board are unaffected; done cards' legacy ledgers
   are never re-validated on any live path (only
   accept/block/resume read ledgers, and no live
   card is in those states).
 - Design core: hash-chaining + snapshot binding +
   prefix-rule legacy tolerance + fail-closed
   placement before mutation is a sound, minimal
   architecture for the stated goal. The two P1s are
   both about underspecified edges (field-set
   strictness, binding-record presence), not about
   the core approach.
