 ### P2-1 — Authorization-denied FAIL outcome still banks playbook learning evidence via the observer

 - Evidence: src/metaharness/core/executor.py:244-255 (denial sets outcome.final_verdict = Verdict.FAIL then break); src/metaharness/core/executor.py:596-598 (if self.observer is not None: ... self.observer(outcome)
   runs unconditionally after the loop, including for a denied outcome); src/metaharness/correction/learning.py:89-92 (if outcome.final_verdict != Verdict.UNVERIFIED: passed = outcome.final_verdict == Verdict.PASS; for
   bullet_id in used: self.playbook.mark(bullet_id, helpful=passed) — marks applied hints helpful=False on the FAIL). FailureStats.observe is clean (it iterates outcome.attempts, which is empty on denial —
   mast.py:40-44), so only the playbook bullet scoring is affected.
 - Impact: In canonical wiring (web/state.py sets observer=self.learning.observe and playbook_hints=self.knowledge_aware_hints, so LearningLoop.hints_for populates self._applied[task.id] before the loop), a denied
   dispatch marks the task's applied playbook hints harmful — a non-causal attribution (an authorization denial says nothing about hint quality), which .agents/meta18-build-spec.md §1 explicitly forbids ("no ...
   learning evidence" on denial). The capability matrix and failure stats remain clean. Low impact: the path only fires for a defective/malicious issuer or a revocation-between-issue-and-check race (the canonical
   TokenIssuer always issues valid tokens, so denials do not occur in normal operation), it is consistent with pre-existing FAIL handling (budget/routing FAILs already mark hints), and the focused denial tests pass
   observer=None so they neither cover nor catch this.
 - Remediation: In TaskExecutor.execute, do not call self.observer(outcome) when the outcome is FAIL due to authorization denial — e.g., set an internal _authorization_denied flag at executor.py:254 and gate the
   observer call at executor.py:596 on it — so a denial produces only the task.authorization_denied provenance record and a FAIL outcome. Add a canonical-wiring denial test that supplies a real observer+playbook and
   asserts no playbook marks are banked ( locking the spec's "no learning evidence" clause currently uncovered by tests/test_executor.py:75).
