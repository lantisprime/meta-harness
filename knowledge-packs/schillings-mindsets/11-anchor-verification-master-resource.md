# Anchor V — cheap verification is the master resource

**Anchor proposition:** progress compounds wherever one honest try-verify
iteration is cheap; verification cost, not domain glamour, decides where
effort pays.

Schillings never argues *for* this — he argues *from* it. Four independent
decisions in the talk derive from it, which is what qualifies it as a
logical thinking anchor rather than a claim:

1. **The 2018 Pitchfork bet on code** (~08:00): "it is a domain where doing
   verification is reasonable — you can run it, compile it, unit-test it.
   That brought us where we are today." V + scrapeable GitHub → code is the
   tractable domain.
2. **The chemistry/biology expansion** (~17:40): "this ability to
   experiment very quickly in code is impacting other domains — doing
   experiments becomes basically free." V + code cheapening their
   verification loops → now tractable too.
3. **The self-play pivot** (~09:49): human training data exhausted, but
   models "can create their own challenges [and] judge the validity of the
   answer." V + generate-what-you-can-judge → self-play works for code.
4. **The compression eval** (~14:35): loss = compressed size + source size —
   chosen precisely because one evaluation stays trivially cheap while
   improvement is unbounded.

Holdout property: V extracted from the code-only sections alone predicts
his chemistry/biology position before he states it — the anchor
re-derives decisions it was not extracted from, the publish criterion for
a logic entry.
