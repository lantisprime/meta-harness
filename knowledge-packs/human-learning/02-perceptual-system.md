# The Perceptual System — categorical relationships

The Perceptual System finds patterns across past environments to infer the present one. It uses **pattern completion** to produce *categorizations of situations* — inferring a larger picture from partial cues (e.g. reading subtle facial expressions to judge whether a social situation is hostile, friendly, or indifferent). By storing categorical relationships, it learns which features of the environment are relevant to the action-selection systems.

Key properties:

- **Perception is active, not passive.** It entails active *inattention* to non-informative features, and information-seeking actions that bring cues into focus. Attention is directed, in part, by feedback from the action-selection systems.
- **Categories are learned, not innate**, and arrive only with extended experience. Examples: Spanish speakers hear the tapped/trilled r distinction English speakers miss; trained physicists perceive symmetries, engineers perceive syntax types, teachers perceive subtle student frustration.
- **Situational categorization changes evaluation and perception.** The same event (chest pain, crying, eye contact) is experienced differently by situation, and which features are even distinguishable depends on the situation.
- It governs **problem recognition** for cognitive skills the same way it governs sensory perception.

**Teaching implications.** Learners train this system when the environment requires a new scheme for classifying contexts, especially when actions are context-sensitive. It learns *faster under decision-making conditions*, so perceptual exercises are best when the categorization is tied to choosing a strategy (e.g. identify the type of symmetry, then pick a solution approach) rather than practiced in isolation. Because perception depends on prototypes **and** boundaries, lessons should show a diverse range of cases rather than drilling one prototype. The corresponding feedback is **"what"** — direct attention to the key situational features and underlying deep structure. Contrasting cases should contrast *relevance to a decision*.
