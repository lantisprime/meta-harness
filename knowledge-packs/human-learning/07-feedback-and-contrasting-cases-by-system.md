# Matching feedback and contrasting cases to each system

Because each system learns from a different information type, **feedback and contrasting cases must be matched to the system being trained**. The paper frames the feedback types informally as the *what, why, when-how,* and *who* of the content:

| System | Feedback ("...") | Contrasting cases contrast | Practice type | Isolation vs integration |
|---|---|---|---|---|
| Perceptual | **What** features indicate the ideal strategy/action | Relevance to a decision | Categorizing situations | Integrated with the initial choice, but without deep practice of other systems |
| Instinctual | **Who** — which internal states predict success/belonging | Experiences good in one situation, bad in others | Stretching beyond one's comfort zone | Both isolated *and* integrated into all other exercises |
| Deliberative | **Why** — underlying causal/structural relationships | Outcomes from a given choice | Navigating knowledge structures | Isolated from procedural exercises |
| Procedural | **When and how** to perform each action correctly | Superficial features | Precise repetition of actions | Isolated from deliberative exercises |

Key design principles that follow:

- **You often cannot optimize all systems in one exercise.** Some learning is best *integrated* (perceptual categorization tied to a decision; instinctual learning woven into everything), while other learning is best given *dedicated, isolated* time (deliberative exploration vs procedural repetition, which actively conflict).
- **Method of contrasting cases**: give multiple similar exercises that draw attention to the key differences appropriate to the target system.
- The *type of assessment must align with the type of instruction.* Feedback that promoted deliberative learning but was assessed procedurally (or vice versa) will underestimate real gains.
- Perceptual feedback should show the **range of generalization** (prototypes and boundaries), not drill a single prototype; procedural feedback should **not** explain causal logic, because that isn't the regularity the Procedural System learns from.
